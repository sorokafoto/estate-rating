# Рейтинг скорости реакции застройщиков на заявки

Одностраничный статический сайт для публичной презентации исследования: интерактивная
сортируемая таблица-рейтинг, вкладка номинаций и CTA-форма.

## Главный принцип — приватность

Сырые событийные данные с PII (телефоны, время событий, ID заявок) **никогда не попадают
в браузер**. Наружу деплоится только обезличенный агрегат `data.json` (метрики уровня
застройщика + название + сайт).

```
private/source.xlsx   (PII, в .gitignore)
        │  npm run build-data
        ▼
   data.json           (только агрегаты + developer_name + url)
        │
        ▼
   статический сайт    (index.html + assets/ + config.js + data.json)
```

## Структура

| Путь | Назначение |
|------|------------|
| `index.html`, `assets/styles.css`, `assets/app.js` | сам сайт (vanilla, без зависимостей) |
| `config.js` | единая конфигурация: путь к данным, акцент, тексты, номинации, endpoint формы |
| `data.json` | публичный агрегат (генерируется) |
| `build/build-data.mjs` | шаг сборки: источник → агрегат → `data.json` + проверка PII |
| `build/source.mjs` | чтение `private/source.xlsx` (лист `events messengers`) |
| `build/aggregate.mjs` | расчёт метрик по застройщику |
| `build/validate.mjs` | защитная regex-проверка отсутствия телефонов/PII |
| `build/applications-sent.json` | знаменатель заявок (default + overrides по застройщику) |
| `private/` | приватный источник (в `.gitignore`, не коммитится) |

## Запуск

```bash
npm install            # ставит xlsx (только для шага сборки)
npm run build-data     # private/source.xlsx -> data.json (+ PII-валидация)
npm run serve          # http://localhost:4321
```

Если `private/source.xlsx` отсутствует — `build-data` сгенерирует демо-данные и пометит
`data.json` флагом `demo: true`.

## Метрики (считаются на сборке, не в браузере)

В метрики идут только события с `identified = да`.

- **Время ответа** — время до первого контакта по любому каналу (на заявку `min(lead_response_time)`),
  усреднённое/медиана по заявкам с откликом. (Звонков в данных почти нет, поэтому метрика
  канало-независимая.)
- **Без ответа, %** — `(N − заявок_с_откликом) / N`, где `N` — число отправленных заявок
  (`applications-sent.json`, по умолчанию 21; реальные значения подставляются позже).
- **Перезвоны/72ч** — сумма событий с `recontact = да` на заявку.
- **Касания/72ч** — все касания (любой канал) на заявку.
- **Маркировка, %** — доля событий с `is_marked = да`.
- **WhatsApp/Telegram/SMS/Max, %** — доля заявок (из `N`) с контактом по каналу.

Нет данных по метрике → `null` (в таблице «—»), не 0.

## Деплой

В хостинг/репозиторий-публикацию попадают только: `index.html`, `assets/`, `config.js`,
`data.json` (и при необходимости `data.js` для открытия без сервера). Папка `private/` и любые
`*.xlsx`/выгрузки/ключи исключены через `.gitignore`.

Заголовки безопасности (CSP и др.): см. `deploy/_headers` (Netlify/Cloudflare Pages) или
`deploy/nginx-security-headers.conf` (nginx). Локально те же заголовки выставляет `npm run serve`.

Контракт метрик: `shared/metrics.mjs` → при сборке генерируются `assets/metrics.js` и
`assets/url-utils.js`. Тесты: `npm test`.

## Ограничения первой версии

- Источник — частичный реальный срез; в таблице столько застройщиков, сколько есть в данных.
- Знаменатель «отправленные заявки» пока допущение (21); у части застройщиков их меньше —
  точные числа подставляются в `applications-sent.json` позже.
- Endpoint CTA-формы не задан → используется mailto-фолбэк (см. `config.js`).
