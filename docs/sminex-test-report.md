# Тестовый расширенный отчёт: Sminex

Дата подготовки: 16 июня 2026  
Период исследования: с 2 по 8 июня 2026 года  
Источник: агрегированные данные публичного рейтинга `developer-response-rating`  
Статус: тестовая версия отчёта, без приложенного реестра сырых заявок и событий

## 1. Короткий вывод

Sminex показал одну из лучших скоростей первого идентифицированного звонка в исследовании: медиана времени до первого такого звонка составила 1 минуту. Это на уровне лучшего значения рынка и значительно быстрее среднего рыночного показателя.

Главная зона риска не скорость, а покрытие заявок: по 45% тестовых заявок за 72 часа не было зафиксировано идентифицированного звонка. При этом в публичных агрегатах нет признаков использования SMS или мессенджеров, поэтому если клиент не берёт трубку или звонок не проходит, сценарий повторного контакта почти не виден.

Если контакт не удалось надёжно идентифицировать, он не засчитывался как ответ Sminex.

## 4. Воронка реакции

| Этап | Количество | Доля от отправленных заявок |
|---|---:|---:|
| Отправлено тестовых заявок | 20 | 100% |
| Был хотя бы один идентифицированный звонок Sminex | 11 | 55% |
| Не было идентифицированного звонка за 72 часа | 9 | 45% |
| Зафиксированы SMS/мессенджеры | 0 | 0% |

Интерпретация: если идентифицированный звонок случается, он происходит очень быстро. Но почти половина заявок не получает идентифицированного звонка в 72-часовом окне. Это может означать как отсутствие звонка, так и то, что контакт не удалось надёжно связать с Sminex.

## 5. Каналы и касания

По текущим агрегатам все идентифицированные контакты Sminex относятся к звонкам. SMS, Max, WhatsApp и Telegram в данных по компании не зафиксированы.

Это не обязательно означает, что компания не использует эти каналы вообще. Корректная формулировка для встречи: в рамках тестовых заявок исследования мы не увидели идентифицированных SMS или сообщений в мессенджерах, связанных с Sminex.

Практический риск: при недозвоне нет видимого альтернативного канала, который мог бы вернуть клиента в диалог. Для проверки стоит сопоставить наши тестовые номера с CRM и телефонией: были ли SMS, автосообщения, WhatsApp-касания или задачи менеджерам, которые не попали в наблюдение.

## 6. Спам и риск утечки заявок

По рынку в целом зафиксировано 7 121 входящее событие, из них 3 006 классифицированы как спам или неидентифицированные контакты. Рыночная доля таких событий в публичных агрегатах составляет 42%.

Для Sminex в текущем публичном `data.js` нет отдельной выгрузки спам-событий по заявкам компании. Поэтому в тестовом отчёте нельзя честно указать:

- сколько заявок Sminex получили спам-контакты;
- какие номера звонили как спамеры;
- кто был быстрее по каждой заявке: Sminex или спамер;
- какие спам-контакты можно передать на проверку службе безопасности или подрядчику по сайту.

Для финального отчёта нужен отдельный сырой лист `events` по заявкам Sminex, где каждое событие связано с `application_id`. Шаблон такого файла добавлен отдельно: [`sminex-events-template.csv`](sminex-events-template.csv). Перед передачей отчёта его нужно заполнить из сырой выгрузки событий только по Sminex.

## 8. Что проверить в CRM Sminex

1. Найти 20 тестовых заявок по номерам, которые будут переданы отдельным реестром.
2. Заполнить отдельный `events` по Sminex из сырой выгрузки и связать каждое событие с `application_id`.
3. Проверить 9 заявок, по которым в исследовании не было зафиксировано идентифицированного звонка за 72 часа.
4. Проверить, были ли по этим заявкам внутренние задачи, статусы, причины недозвона или антиспам-блокировки.
5. Проверить, уходили ли SMS или сообщения в мессенджеры после заявки или после первого недозвона.
6. Сравнить время первого идентифицированного звонка в телефонии Sminex с временем, зафиксированным исследованием.
7. Отдельно проверить, не попадали ли тестовые номера в спам-фильтры, автоопределители или блокировки оператора.

## 9. Рекомендации

Сохранить сильную сторону: мгновенный первый звонок там, где заявка попадает в рабочий сценарий. Медиана в 1 минуту — показатель, который стоит использовать как внутренний стандарт.

Разобрать причины непокрытых заявок. 45% без идентифицированного звонка — главный управленческий вопрос: форма, интеграция, распределение в CRM, телефония, антиспам-фильтр, график работы или правила обработки повторных/тестовых номеров.

Добавить сценарий повторного контакта после первого недозвона: SMS или мессенджер с коротким сообщением, кто звонил и как выбрать удобное время. В исследовании такой сценарий по Sminex не виден.

Сделать регулярную сверку «заявка на сайте -> лид в CRM -> первый звонок -> повторное касание». Это позволит отличать проблему скорости от проблемы доставки заявки.

## 10. Приложение: структура сырых данных для финальной версии

Текущий тестовый отчёт не содержит строки сырых заявок и событий, потому что они не представлены в публичной сборке проекта. Для финальной передачи Sminex нужен XLSX с такими листами.

### `applications`

| Поле | Описание |
|---|---|
| `application_id` | Уникальный ID тестовой заявки |
| `developer_name` | Sminex |
| `site_url` | Сайт или страница формы |
| `submitted_at` | Дата и время отправки заявки |
| `submitted_time_slot` | Утро, день или вечер |
| `test_phone_number` | Номер, на который ожидали ответ |
| `form_status` | Отправлена или ошибка |
| `form_issue` | Описание проблемы формы, если была |

### `events`

| Поле | Описание |
|---|---|
| `event_id` | Уникальный ID события |
| `application_id` | Связь с заявкой |
| `developer_name` | Sminex |
| `site_url` | `sminex.com` |
| `application_submitted_at` | Дата и время исходной заявки |
| `event_at` | Дата и время контакта |
| `minutes_since_application` | Время от заявки до события |
| `channel` | call, sms, max, whatsapp, telegram |
| `from_phone` | Номер, с которого был контакт |
| `identified_status` | developer, spam, unknown, agency |
| `identified_developer` | Sminex только для проверенных контактов компании |
| `verification_source` | Источник идентификации |
| `verification_confidence` | high, medium, low |
| `is_first_identified_company_call` | Первый проверенный звонок Sminex по заявке |

### `spam_contacts`

| Поле | Описание |
|---|---|
| `application_id` | Связь с заявкой Sminex |
| `spam_event_at` | Дата и время спам-контакта |
| `from_phone` | Номер спамера |
| `phone_mark` | Маркировка или подпись номера |
| `spam_category` | Риелтор, агентство, робот, неизвестно |
| `spam_beat_company` | Был ли спам быстрее первого идентифицированного звонка Sminex |

## 11. Ограничения тестовой версии

В текущих данных нет дат и времени отдельных заявок, тестовых номеров, строк событий и спам-контактов по Sminex. Поэтому этот документ показывает формат управленческого отчёта и использует только проверяемые агрегаты из публичной сборки.

Для полноценной передачи застройщику нужно подключить внутреннюю сырую выгрузку исследования и автоматически заполнить приложения `applications`, `events` и `spam_contacts`.
