# Каталог метрик рейтинга

Рабочий документ для согласования с руководством: какие метрики показывать в таблице на сайте, по каким строить номинации, что выводить на вкладке «По рынку», а что не публиковать вообще.

**Как пользоваться:** колонка **«Сейчас»** — что уже на сайте (baseline). Колонки **«Таблица / Номинация / Скрыть»** — отметьте желаемое состояние после согласования.

> **Данные обновляемые.** Срез на 2026-06-13; после доработки phone book цифры «Без ответа» и каналы могут сдвинуться. В `data.json` стоит флаг `partial: true`.

## Как считаем (общее)

- **N** — сколько заявок **фактически отправили** застройщику (по листу `applications`; у разных участников может быть 17, 20, 21 и т.д.).
- **R** — сколько заявок получили хотя бы один контакт за 72 часа («заявки с откликом»).
- Событие учитывается в метриках, только если `identified = да`, привязано к заявке (`application_id`) и попадает в окно **72 часа** после заявки (`lead_response_time` 0–4320 мин).
- «Первый контакт» по заявке = минимальное `lead_response_time` среди её событий.

Подробнее о правилах участия — раздел **H**. О границе публичности — раздел **G**.

## Статус данных (важно)

SMS и звонки уже сматched с заявками и входят в рейтинг. Мессенджеры (Max / WhatsApp / Telegram) в выборке почти нули.

| Канал | Идентификация | Матч с заявками | В рейтинге |
|-------|---------------|-----------------|------------|
| SMS | да (по маркировке) | да | да |
| Звонки | частично (phone book дорабатывается) | да | да |
| Max / WhatsApp / Telegram | мало данных | частично | почти нули |

**Легенда колонки «Данные»:**

- **Есть** — метрика заполнена в текущем `data.json`.
- **Не реализовано** — в каталоге для обсуждения, агрегатор (`aggregate.mjs`) пока не считает.
- **TBD** — теоретически можно посчитать из source, нужна проверка полей перед обещанием руководителю.

## A. Скорость реакции

| Метрика | Формула / смысл | Тип | Данные | Сейчас | Таблица | Номинация | Скрыть |
|---|---|---|---|:--:|:--:|:--:|:--:|
| Ср. скорость ответа | медиана `lead_response_time` по заявкам с откликом (в UI — «ср.» для простоты) | время | Есть | **да** | ☐ | ☐ | ☐ |
| Место в рейтинге (#) | порядковый номер по сортировке (default: `avg_response` asc) | ранг | Есть | **да** | ☐ | ☐ | ☐ |
| Самый быстрый ответ | минимальное время первого контакта среди всех заявок застройщика | время | Есть | нет | ☐ | ☐ | ☐ |
| Доля быстрых, ≤15 мин | заявки с первым контактом ≤15 мин / R | % | Не реализовано | нет | ☐ | ☐ | ☐ |

> Номинация «Самый быстрый ответ» на сайте использует **медиану** (`avg_response`), а не минимум по заявкам.

## B. Охват (дозвон)

| Метрика | Формула / смысл | Тип | Данные | Сейчас | Таблица | Номинация | Скрыть |
|---|---|---|---|:--:|:--:|:--:|:--:|
| Без ответа, % | (N − R) / N × 100 | % | Есть* | **да** | ☐ | ☐ | ☐ |
| Ответили, % | R / N × 100 (дубль «Без ответа») | % | Есть* | нет | ☐ | ☐ | ☐ |
| Недостаточно данных | статус при N < 11; метрики не считаются | статус | Есть | **да** | ☐ | ☐ | ☐ |

\* **Caveat идентификации (раздел I):** 100% «без ответа» может означать и отсутствие звонка, и то, что контакт не удалось идентифицировать. После обновления phone book доля может снизиться у части застройщиков.

## C. Объём и настойчивость

| Метрика | Формула / смысл | Тип | Данные | Сейчас | Таблица | Номинация | Скрыть |
|---|---|---|---|:--:|:--:|:--:|:--:|
| Касания, всего | сумма всех контактов по всем заявкам (`total_touches`) | абс | Есть | **да** | ☐ | ☐ | ☐ |
| Касаний на заявку | сумма контактов / **R** | число | Не реализовано | нет | ☐ | ☐ | ☐ |
| Повторные касания | `sum(max(0, контактов−1)) / **N**` — на все отправленные заявки, включая без ответа | число | Есть | **да** | ☐ | ☐ | ☐ |
| Повторные звонки | сумма (звонков по заявке − 1) | абс | Не реализовано | нет | ☐ | ☐ | ☐ |
| Звонков всего / на заявку | число звонков | абс / число | Не реализовано | нет | ☐ | ☐ | ☐ |
| «Не отпускают» (рекорд) | макс. число касаний по одной заявке | абс | Не реализовано | нет | ☐ | ☐ | ☐ |

Повторы считаются из событий, без служебного флага `recontact`. «Повторные звонки» как отдельная номинация слабо различают участников — лучше «повторные касания» или рекорд «Не отпускают».

> На сайте **«Повторные касания»** делятся на **N**, а не на R. «Касаний на заявку» (/ R) в коде нет — это кандидат на добавление.

|---|---|---|---|:--:|:--:|:--:|:--:|
| Звонок, % | заявки со звонком / N × 100 | % | Есть | **да** | ☐ | ☐ | ☐ |
| SMS, % | заявки с SMS / N × 100 | % | Есть | **да** | ☐ | ☐ | ☐ |
| Max, % | заявки с Max / N × 100 | % | Есть (≈0) | **да** | ☐ | ☐ | ☐ |
| WhatsApp, % | заявки с WhatsApp / N × 100 | % | Есть (≈0) | **да** | ☐ | ☐ | ☐ |
| Telegram, % | заявки с Telegram / N × 100 | % | Есть (≈0) | **да** | ☐ | ☐ | ☐ |
| Мессенджеры, % (свёрнуто) | заявки с любым мессенджером / N × 100 | % | Есть (≈0) | нет | ☐ | ☐ | ☐ |

## F. Уровень рынка (не по застройщику)

**Правило:** вкладка «По рынку» — только **рыночная аналитика без имён**. Сравнение двух застройщиков и топ-3 — **только «Номинации»**.

Бенчмарк считается по застройщикам **с кворумом** (`insufficient_data = false`). В `data.json` → `market.sample_size` (сейчас 97 из 100). На карточках **нет строки «Лучший на рынке»**.

### Карточки вкладки «По рынку» (6 шт.)

| # | Карточка | Формула / смысл | Поле в data.json | Сейчас |
|---|----------|-----------------|------------------|:------:|
| 1 | Заявок без ответа | среднее `no_callback_share` по выборке | `market.no_callback_share.mean` | **да** |
| 2 | Застройщиков без единого ответа | count `no_callback_share === 100` | `market.silent_developers_count` | **да** |
| 3 | До первого контакта | среднее `avg_response` (медиан по компаниям с откликом) | `market.avg_response.mean` | **да** |
| 4 | Ответ дольше суток | count `avg_response > 1440` мин | `market.slow_response_count` | **да** |
| 5 | Входящих — не застройщик | `identified ≠ да` / все входящие | `market.spam_share.mean` | **да** |
| 6 | Контакт в мессенджере | среднее суммы WA+TG+Max; разбивка по каналам | `market.messengers` | **да** |

Подпись под блоком (динамически): *100 застройщиков · 2 001 заявка · окно 72 часа · агрегаты по 97 компаниям…*

### Номинации (отдельная вкладка, 6 шт. — с именами, топ-3)

| Номинация | Метрика / логика | Сейчас | Номинация | Скрыть |
|---|---|:--:|:--:|:--:|
| Самый быстрый ответ | min `avg_response` (медиана), top-3 | **да** | ☐ | ☐ |
| Самый настойчивый | max `avg_recontacts`, top-3 | **да** | ☐ | ☐ |
| Больше всего касаний | max `total_touches`, top-3 | **да** | ☐ | ☐ |
| Самый омниканальный | max числа мессенджерных каналов, top-3 | **да** | ☐ | ☐ |
| Чемпион мессенджеров | max суммы WA+TG+Max, top-3 | **да** | ☐ | ☐ |
| «Бульдожья хватка» | рекорд касаний по одной заявке (max touches_per_app) | **да** | ☐ | ☐ |
| Самые обязательные | min `no_callback_share` (самая высокая конверсия в дозвон) | нет | ☐ | ☐ |
| Работают в выходные | первые контакты в сб/вс | нет | ☐ | ☐ |

> **Разрыв min/max по скорости** (дельта между лидером и аутсайдером) **не публикуется** на сайте — ни в номинациях, ни на «По рынку». Парное сравнение двух застройщиков по имени некорректно для публичного инфоповода; при необходимости — только в пресс-ките без имён или во внутренней аналитике.

## G. Публичность: что показываем, что нет

| Данные | Публично на сайте | По запросу застройщика (CTA) | Никогда не публикуем |
|--------|:-----------------:|:----------------------------:|:--------------------:|
| Название застройщика, домен сайта | да | — | — |
| Агрегаты: скорость, без ответа, касания, каналы | да | детализация по компании | — |
| Место в рейтинге, номинации, рыночные средние | да | сравнение с рынком | — |
| Число отправленных заявок (`applications_sent`) | в JSON, не в таблице | — | — |
| Статус «Недостаточно данных» | да | — | — |
| Meta: 100 застройщиков, 2000+ заявок, период | да (hero / meta) | — | — |
| Даты и время контактов по каждой заявке | нет | да | — |

## H. Правила участия в рейтинге (не метрика, но влияет на таблицу)

- **Кворум:** минимум **11** успешно отправленных заявок (`N_actual ≥ floor(21 / 2) + 1`).
- При **N < 11:** статус «Недостаточно данных», все метрики = `null`, место = «—», номинации и рыночный бенчмарк не включают застройщика.
- **Сортировка по умолчанию:** по `avg_response` (быстрее — выше); строки без кворума — в конце таблицы.
- **Top-3:** подсветка первых трёх мест (только среди участников с кворумом).

## I. Caveats и риски (для согласования)

### Идентификация и «Без ответа, %»

- Если у застройщика 100% без ответа — это может быть и реальное молчание, и **невозможность связать звонок** с застройщиком (нет маркировки, неподписанные SMS, неотвеченный обратный прозвон).
- Идентификация звонков **ещё в работе**; после обновления phone book часть застройщиков может подняться в рейтинге.
- На сайте есть блок «Важно об идентификации» в методологии — формулировку стоит согласовать вместе с метрикой B.

### Риски при публикации

- **Звонок, %** у многих ≈50% — шум в таблице, слабый дифференциатор.
- **Касания (абсолют)** и **Повторные касания** частично дублируют «настойчивость».
- Номинация **«Больше всего касаний»** коррелирует с объёмом ответов, а не с качеством процесса.
- **«Самый омниканальный»** и **«Чемпион мессенджеров»** при текущих данных почти не различают участников (мессенджеры ≈0).
- **`partial: true`** — первая публичная версия; цифры обновятся после доработки данных.

## Черновая рекомендация (для лаконичной таблицы)

Без дублей «абсолют/процент» и без пустых колонок:

- **Таблица:** Ср. скорость ответа · Без ответа, % · Касаний на заявку (/ R) · Звонок, % · SMS, % (мессенджеры — свернуть в «Мессенджеры, %» или скрыть, пока ≈0).
- **Номинации:** Самый быстрый · Самый настойчивый · Бульдожья хватка · Самые обязательные · Работают в выходные.
- **Скрыть / только в номинации:** абсолютные «всего», время суток, отдельные мессенджер-каналы, «Больше всего касаний» (абсолют), «Омниканальный», «Чемпион мессенджеров».

### Расхождение: сайт шире рекомендации

| Зона | Сейчас на сайте | Черновая рекомендация |
|------|-----------------|----------------------|
| Таблица | 10 колонок метрик + ранг + статус | ~5–6 колонок |
| Номинации | 5 шт. | 3–4 шт. |
| «По рынку» | **6 карточек** (PR-инфоповоды, без имён) | см. раздел F |

> Почему убрали «Дожимают звонком»: звонок — основной канал почти у всех, повторные звонки делают практически все застройщики. Номинация не выделяет лидера — это «Самый настойчивый» (повторные касания).
>
> Та же логика про «Звонок, %» и «Омниканальный»: после матчинга звонков разброс по «Звонок, %» оказался небольшим (~50% у многих) — колонку и номинацию можно убрать.

> **Финальный выбор — за вами и руководством.** Отметьте галочки в колонках «Таблица / Номинация / Скрыть», и сайт приведут в соответствие (`shared/metrics.mjs`, `config.js`).
